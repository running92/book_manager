# 图书管理系统毕业设计答辩讲稿

## 1. 项目介绍

各位老师好，我的毕业设计题目是 **《图书管理系统的设计与实现》**，英文名称是 **Library Management System**。

本项目主要面向中小型图书管理场景，系统分为 **管理员** 和 **普通读者** 两类角色。管理员可以对图书、分类、用户和借阅记录进行统一管理；普通读者可以进行图书查询、图书借阅、查看个人借阅记录等操作。

在技术实现上，我采用了前后端分离的开发方式。前端使用 **Vue 3 + Vite** 构建单页应用，后端使用 **Python Flask** 提供 RESTful API，数据库采用 **SQLite**。同时，为了提高系统展示效果和实用性，系统支持 **中文和英文双语切换**，适合毕业设计展示和扩展开发。

本项目的设计目标主要有三点：

第一，完成一个具有实际业务闭环的图书管理系统；  
第二，实现从用户认证、图书管理到借阅归还的完整流程；  
第三，在功能完整的基础上，兼顾界面展示效果、部署便捷性以及答辩演示友好性。

---

## 2. 项目意义

本课题具有一定的实际意义和应用价值。

首先，在很多中小型图书室、资料室或学习场景中，仍然存在图书登记效率低、借阅记录不清晰、人工管理繁琐等问题。本系统通过信息化手段，对图书、用户和借阅流程进行统一管理，可以有效提升管理效率。

其次，从软件工程实践角度看，本项目涵盖了用户认证、权限控制、数据库建模、前后端交互、国际化设计等多个典型模块，能够较完整地体现一个管理系统从需求分析到设计实现的全过程。

最后，本系统采用轻量化技术方案，部署简单、运行方便，具有较好的演示价值和后续扩展价值。

---

## 3. 技术选型

本系统主要采用以下技术：

前端方面，使用 **Vue 3** 作为核心框架，结合 **Vite** 进行工程构建。之所以选择 Vue 3，是因为其组件化开发模式较为清晰，适合中小型后台管理系统开发；而 Vite 启动快、打包效率高，适合前端工程化实践。

在 UI 层面，使用 **Element Plus** 构建系统页面，以提高界面开发效率和整体美观度。

在状态管理方面，使用 **Pinia** 管理登录信息、用户权限和语言状态；在国际化方面，使用 **vue-i18n** 实现中英文双语切换。

后端方面，选择 **Flask** 框架。Flask 具有轻量、灵活、易于扩展的特点，适合毕业设计类项目快速实现完整后端服务。数据库使用 **SQLite**，主要考虑到其轻量化、易部署、无需单独安装数据库服务，便于本地运行和答辩演示。

另外，系统登录认证采用 **JWT**，提升了前后端分离场景下的认证灵活性与安全性。

## 4. 系统功能概述

本系统主要分为以下几个功能模块：

### 4.1 登录与权限管理模块

用户通过用户名和密码登录系统，系统根据用户角色区分管理员和普通读者，并进行不同菜单和接口权限控制。

### 4.2 系统首页模块

管理员首页主要展示图书总数、可借数量、借阅记录数、用户数量等统计信息；普通用户首页则展示推荐图书和个人借阅情况。

### 4.3 图书管理模块

管理员可以对图书进行新增、编辑、删除、查询等操作；普通用户可以查看图书列表和图书详情。

### 4.4 分类管理模块

管理员可以维护图书分类，支持分类的新增、编辑和删除，同时系统会限制删除仍有关联图书的分类。

### 4.5 用户管理模块

管理员可以管理系统用户，包括新增用户、编辑资料、禁用用户和重置密码等。

### 4.6 借阅管理模块

普通用户可借阅图书并查看自己的借阅记录；管理员可以查看全部借阅记录并进行归还处理。系统支持库存校验、归还更新、逾期状态判断等业务逻辑。

### 4.7 个人中心模块

用户可以查看并修改自己的基础信息，修改密码，并查看个人借阅历史。

## 5. 项目亮点

本项目的亮点主要体现在以下几个方面：

### 5.1 支持中英双语

本系统不仅实现了基本业务功能，还支持中文和英文双语切换。页面菜单、按钮、提示语、表单文本等都可以根据语言动态切换，这使系统在展示层面更完整，也更具有扩展价值。

### 5.2 采用 SPA 单页应用架构

前端基于 Vue 3 + Vite 构建成单页应用，页面切换流畅，用户体验较好。系统打包后可直接部署，适合毕业设计演示。

### 5.3 前后端分离，结构清晰

前端负责页面展示与交互，后端负责业务逻辑和数据接口，系统层次清晰，便于维护和扩展，也符合当前主流 Web 系统开发模式。

### 5.4 业务闭环完整

系统并不仅仅停留在静态页面展示，而是真正实现了从图书管理、用户管理，到借书、还书、逾期判断的完整业务闭环。

### 5.5 部署简单，演示友好

系统采用 SQLite 数据库，无需额外安装数据库服务；同时 Flask 可以直接托管前端打包后的静态资源，方便本地启动和答辩展示。

## 6. 系统演示讲稿

接下来我简单演示一下系统功能。

首先进入登录页面。这里可以选择使用管理员账号或者普通用户账号登录。登录成功后，系统会根据角色显示不同的菜单和功能模块。

管理员登录后，可以看到系统首页，首页上展示了图书总数、当前可借图书数量、用户总数以及借阅记录统计等信息。

然后进入图书管理模块，可以看到图书列表。这里支持按照书名、作者和分类进行查询，也支持新增、编辑和删除图书。每本图书都包含中文标题和英文标题，支持双语展示。

在分类管理模块中，管理员可以维护图书分类。如果分类下仍有关联图书，系统会阻止删除，保证数据完整性。

在用户管理模块中，管理员可以查看所有用户信息，并进行新增用户、编辑用户、禁用用户和重置密码等操作。

接下来进入借阅管理模块。这里可以查看所有借阅记录，包括借阅中、已归还和已逾期等状态。当普通用户借书时，系统会自动校验库存是否足够；当管理员执行归还操作时，系统会同步更新借阅记录状态以及图书可借库存。

最后切换到普通用户账号。普通用户可以查看图书列表、图书详情、借阅图书，并在“我的借阅”页面中查看个人借阅记录。

此外，页面右上角提供了中英文切换按钮，点击后整个系统界面文案会即时切换，这也是本项目的一个展示亮点。

## 7. 项目难点与解决方案

在项目实现过程中，我认为主要有以下几个难点：

### 7.1 权限控制

系统有管理员和普通用户两种角色，不同角色访问的页面和接口不同。  
我的解决方案是：前端通过路由守卫和菜单动态渲染实现页面级权限控制，后端通过 JWT 和角色校验实现接口级权限控制。

### 7.2 借阅业务逻辑

借阅业务看似简单，但涉及库存更新、重复借阅校验、归还处理和逾期状态判断。  
我的解决方案是：将借阅和归还逻辑封装到后端服务层中，统一处理库存扣减、状态变更和时间字段更新，避免逻辑分散。

### 7.3 中英双语支持

系统不仅要显示双语文本，还要保证切换后页面布局正常。  
我的解决方案是：前端统一使用 vue-i18n 管理文案，将页面标题、菜单、按钮、表单、提示语等全部纳入国际化管理。

### 7.4 SPA 打包与部署

由于系统是单页应用，打包部署时需要考虑前后端联调和静态资源托管。  
我的解决方案是：开发环境下通过 Vite 代理转发接口请求；生产环境下通过 Flask 直接托管前端打包后的静态文件，简化部署流程。

## 9. 答辩常见问题与参考回答

### 9.1 为什么选 SQLite，而不是 MySQL？

答：  
因为本项目属于毕业设计，重点是系统设计与功能实现。SQLite 轻量、部署简单、无需额外安装服务，更适合本地开发和答辩展示。对于本系统这种中小规模数据场景，SQLite 已经足够使用。如果后续要扩展到更大规模场景，也可以平滑迁移到 MySQL。

### 9.2 为什么要做中英双语？

答：  
一方面是为了增强系统完整性和展示效果，另一方面也是为了体现系统的可扩展性。通过引入前端国际化机制，可以更方便地适配多语言场景，这也是现代 Web 系统中比较常见的需求。

### 9.3 为什么采用前后端分离？

答：  
前后端分离可以让前端专注界面和交互，后端专注业务和数据接口，结构更清晰，维护性更好，也更符合当前主流开发模式。

### 9.4 系统如何保证借阅库存正确？

答：  
借书时后端会先校验图书状态和可借库存，只有库存大于 0 时才允许借阅；借阅成功后会同步减少 `available_stock`；归还时再增加 `available_stock`。这样可以保证库存数量与借阅状态一致。

### 9.5 密码是怎么存的？

答：  
密码不是明文存储，而是通过哈希加密后保存到数据库中，提高安全性。
